# Новая архитектура: ядро для быстрой разработки и AI‑дружественный код

## 1. Введение

Этот проект представляет собой микро‑фреймворк для создания локальных веб‑приложений на Flask и SQLite с минимальным объёмом кода при добавлении новых возможностей. Основные цели:

- **Декларативность** – новая страница описывается в несколько строк, всё остальное генерируется автоматически.
- **Понятность** – код читается как спецификация, без «магии».
- **AI‑дружественность** – благодаря единообразию и предсказуемости нейросети легко достраивают функциональность без галлюцинаций.
- **Локальность** – все компоненты работают на вашем компьютере, без интернета и внешних библиотек (кроме Flask).

## 3. Ядро (core)

### 3.1. `core/models.py` – описание сущностей

Базовый класс `Entity` и его метакласс автоматически регистрируют все сущности.  
Для создания новой страницы вы наследуетесь от `Entity` и определяете поля:

```python
from core.models import Entity, Field, FieldType

class MyEntity(Entity):
    __tablename__ = 'my_entities'     # имя таблицы в БД
    __page_name__ = 'myentities'      # часть URL (/myentities)
    __title__ = 'Мои сущности'        # заголовок страницы

    fields = [
        Field('name', FieldType.STRING, required=True, label='Название'),
        Field('count', FieldType.INTEGER, label='Количество', default=0),
        Field('date', FieldType.DATE, default='today'),   # default может быть строкой или callable
        Field('is_active', FieldType.BOOOLEAN, default=True),
        # ... другие поля
    ]
```

**Типы полей** (FieldType):
- `INTEGER`, `FLOAT`, `STRING`, `TEXT`, `DATE`, `DATETIME`, `BOOLEAN`, `JSON`.

**Параметры Field**:
- `name` – имя поля (должно совпадать с ключом в данных).
- `field_type` – тип.
- `required` – обязательное ли.
- `default` – значение по умолчанию (может быть функцией).
- `label` – отображаемое название.
- `choices` – список допустимых значений для строкового поля (выпадающий список).
- `**kwargs` – дополнительные атрибуты (например, `min`, `max` для валидации).

После описания сущности она автоматически создаст таблицу в БД (при первом запуске).

### 3.2. `core/db.py` – работа с БД

Класс `Database` предоставляет методы:
- `insert(entity)`
- `update(entity)`
- `delete(entity_cls, id)`
- `get(entity_cls, id)`
- `list(entity_cls, filters=None, order_by=None, limit=None, offset=None)`
- `count(entity_cls, filters=None)`

Все методы работают с объектами `Entity` и автоматически преобразуют типы.

### 3.3. `core/api.py` – автоматический REST API

Функция `register_entity_blueprint(app, entity_cls, db)` создаёт blueprint с эндпоинтами:

- `GET /api/<page_name>/list` – список с фильтрацией, пагинацией.
- `GET /api/<page_name>/item/<id>` – получить одну запись.
- `POST /api/<page_name>/create` – создать.
- `PUT /api/<page_name>/update/<id>` – обновить.
- `DELETE /api/<page_name>/delete/<id>` – удалить.

Все эндпоинты возвращают JSON с полем `status` и `data`/`message`.

### 3.4. `core/ui.py` – автоматический интерфейс

Функция `register_ui_route(app, entity_cls, db)` регистрирует маршрут `/<page_name>`, который отдаёт сгенерированную HTML‑страницу с CRUD‑интерфейсом (таблица + модальное окно для добавления/редактирования). Страница автоматически подстраивается под поля сущности (типы, выпадающие списки, чекбоксы).

Если нужен кастомный интерфейс, можно создать свой шаблон в папке `pages/<page_name>/templates/` и переопределить маршрут вручную.

### 3.5. `core/ml.py` – нейросети и генетические алгоритмы

#### 3.5.1. `SimpleNeuralNetwork`

Двухслойная нейросеть с сигмоидой.  
**Использование**:
```python
from core.ml import SimpleNeuralNetwork

# создание
nn = SimpleNeuralNetwork(input_size=10, hidden_size=5, output_size=3, learning_rate=0.1)

# обучение (X_train – список списков, y_train – список списков)
nn.train(X_train, y_train, epochs=1000)

# предсказание
pred = nn.predict([0.1, 0.2, ...])
```

Методы `save(filepath)` и `load(filepath)` позволяют сохранять/загружать веса.

#### 3.5.2. `GeneticOptimizer`

Генетический алгоритм для оптимизации произвольной функции.  
**Использование**:
```python
from core.ml import GeneticOptimizer

def fitness(params):
    # params – список параметров
    # возвращает число (чем больше, тем лучше)
    return -sum(p**2 for p in params)  # пример: минимизация суммы квадратов

optimizer = GeneticOptimizer(population_size=50, genes=3, fitness=fitness, mutation_rate=0.1, crossover_rate=0.7)
best_params = optimizer.run(generations=100)
print("Best:", best_params)
```

### 3.6. `core/rules.py` – экспертная система

Класс `RuleEngine` позволяет добавлять правила и выполнять вывод.

**Добавление правила**:
```python
engine = RuleEngine()
# способ 1: передать функции условия и действия
def condition(facts):
    return facts.get('state') == 'SICK'
def action(facts):
    facts['recommendation'] = 'Отдых'
engine.add_rule(condition, action)

# способ 2: из строки (синтаксис: IF условие THEN действие)
engine.add_rule_string("IF state == 'SICK' THEN recommendation = 'Отдых'")
```

**Вывод**:
```python
facts = {'state': 'SICK', 'day': 5}
result = engine.infer(facts)
print(result)  # {'state': 'SICK', 'day': 5, 'recommendation': 'Отдых'}
```

Правила выполняются циклически до тех пор, пока факты меняются.

## 4. Как добавить новую страницу (фичу)

### Шаг 1. Создать папку страницы

В `pages/` создайте подпапку, например `mytracker`.

### Шаг 2. Написать модель

Создайте файл `pages/mytracker/model.py`:

```python
from core.models import Entity, Field, FieldType
from datetime import date

class MyTracker(Entity):
    __tablename__ = 'my_tracker'
    __page_name__ = 'mytracker'
    __title__ = 'Мой трекер'

    fields = [
        Field('date', FieldType.DATE, default=lambda: date.today().isoformat(), label='Дата'),
        Field('value', FieldType.FLOAT, required=True, label='Значение'),
        Field('note', FieldType.TEXT, label='Заметка'),
    ]
```

### Шаг 3. Зарегистрировать в `app.py`

Добавьте импорт и регистрацию:

```python
from pages.mytracker.model import MyTracker
register_entity_blueprint(app, MyTracker, db)
register_ui_route(app, MyTracker, db)
```

### Шаг 4. Запустить приложение

При первом запуске таблица создастся автоматически. Перейдите по адресу `/mytracker` – вы увидите готовую страницу со списком, добавлением и редактированием.

### Дополнительные возможности

- **Кастомный шаблон**: создайте `pages/mytracker/templates/mytracker.html` и определите там свой дизайн. В коде `core/ui.py` сначала проверяется существование кастомного шаблона.
- **Кастомный API**: если нужно добавить специфический эндпоинт, вы можете создать blueprint вручную и зарегистрировать его отдельно. Но чаще всего автоматического API достаточно.

## 5. Использование ML и правил внутри страницы

Чтобы интегрировать, например, нейросеть для предсказания, вы можете:

1. В `model.py` добавить метод класса или функцию, которая использует `core.ml`.
2. Вызвать её из обработчика кастомного API (если вы добавили его вручную).
3. Или вызвать из JavaScript на фронтенде (через отдельный AJAX‑запрос).

Пример: добавить эндпоинт `/api/mytracker/predict`, который принимает данные и возвращает предсказание.

В `app.py` после регистрации основного blueprint:

```python
from core.ml import SimpleNeuralNetwork

@bp.route('/predict', methods=['POST'])
def predict():
    data = request.json
    # загрузить модель (например, из файла или из БД)
    nn = SimpleNeuralNetwork(...)
    nn.load('model.json')
    result = nn.predict([data['input1'], data['input2']])
    return jsonify({'prediction': result})
```

## 6. Лучшие практики для AI (нейросетей и людей)

Чтобы нейросеть (или другой разработчик) могла легко расширять проект, соблюдайте следующие принципы:

1. **Используйте декларативное описание сущностей** – не пишите сырые SQL‑запросы и HTML‑шаблоны, если этого можно избежать. Всё, что выходит за рамки автоматики, оформляйте как отдельные модули с чёткими интерфейсами.
2. **Следуйте соглашениям об именах**:
   - Имена классов сущностей в единственном числе (Expense, Task).
   - Поля – в нижнем регистре с подчёркиваниями.
   - Имена файлов – в нижнем регистре.
3. **Минимизируйте переопределение** – если вам нужно изменить поведение, предпочтите расширение через наследование, а не замену файлов ядра. Например, можно создать свою версию `ui.py` в папке страницы, которая переопределяет только нужные части.
4. **Документируйте нестандартные решения** – если вы добавили кастомный код, поясните его назначение в комментариях.
5. **При генерации кода нейросетью**:
   - Предоставьте ей этот файл объяснения.
   - Укажите, что нужно создать только `model.py` и зарегистрировать страницу.
   - Если нужно добавить ML или правила, покажите пример использования соответствующих классов.
   - Избегайте генерации длинных HTML/JS строк – доверьтесь автоматике.

## 8. Часто задаваемые вопросы и советы

**Вопрос:** Могу ли я добавить поля с валидацией, например `min`/`max`?
**Ответ:** Да, в `Field` можно передать любые дополнительные параметры. Валидацию можно реализовать, переопределив метод `validate` в своём классе сущности.

**Вопрос:** Как добавить связь между сущностями (например, один ко многим)?
**Ответ:** Пока автоматика не поддерживает связи, но вы можете добавить поле типа `INTEGER` (внешний ключ) и вручную организовать выборку. В будущем можно расширить ядро.

**Вопрос:** Как добавить сложную бизнес‑логику при сохранении?
**Ответ:** В методе `save` сущности (или в `create`/`update` API) вы можете добавить свои проверки. Например, переопределить `insert` в классе сущности.

**Вопрос:** Как загрузить данные из внешнего источника?
**Ответ:** Создайте скрипт, который использует `Database` и `Entity` для вставки.

**Вопрос:** Как обновить ядро, если вышли новые версии?
**Ответ:** Поскольку ядро – это ваш код, вы можете вносить изменения напрямую. Старайтесь сохранять обратную совместимость.
